# UX Before / After

## Before

- repeated issue/system-warning messaging in default and expanded views
- no compact check-coverage summary
- technical payload visible too early
- file-access/system failures looked too similar to document-quality failures

## After

- one primary issue in the row
- one blocker chip
- one next-action chip
- direct `View`, `Download`, `Re-Verify` actions on uploaded documents
- `Details` remains collapsed by default
- verification coverage summary added inside `Details`
- full check payload moved into nested `Technical audit details`
- runtime-executable policies now show missing expected checks when coverage is incomplete
