# UI Before / After

## Before

- Default document rows exposed too many action buttons and repeated status wording.
- Expanded details repeated the same failure/system-issue message multiple times.
- Secondary case actions remained exposed directly in the top action bar.

## After

- Default rows are compact and closer to the Manus-style review pattern.
- `View` / `Download` stay direct, while secondary actions move into `More`.
- Missing rows support slot-aware `Upload`.
- Technical details prioritize `Material findings` and keep passed checks collapsed.
- Top action bar keeps only primary actions visible.
